# AI Guidance

This project was developed with assistance from AI coding tools to accelerate development while maintaining human oversight.

## Purpose of AI Usage

AI tools were used to assist with:

* Generating boilerplate backend and frontend code
* Debugging frontend–backend API integration
* Improving error handling
* Writing documentation and README sections
* Suggesting improvements to architecture and project structure

AI was used as a **development assistant**, not as an autonomous agent.

---

## AI Tools Used

Examples of tools used during development include:

* AI coding assistants
* prompt-based code generation tools
* AI debugging suggestions

All generated code was reviewed and tested before being committed.

---

## Prompting Strategy

Prompts were written to solve specific engineering tasks such as:

* generating REST API endpoints
* fixing React ↔ Flask connectivity issues
* implementing task extraction logic
* improving project documentation

Prompts were intentionally constrained to avoid large uncontrolled code changes.

---

## Human Review Process

All AI-generated outputs were:

1. Reviewed manually
2. Tested locally
3. Modified if necessary before integration

This ensured the system remained understandable and maintainable.

---

## Constraints Applied to AI

The following rules were followed when using AI assistance:

* Do not rewrite the entire project automatically
* Maintain clear separation of backend and frontend layers
* Preserve readable and simple code
* Prefer small incremental changes over large generated blocks

---

## Coding Standards

The project follows these principles:

* RESTful API design
* Separation of concerns (routes, services, models)
* Minimal dependencies
* Clear project structure
* Simple relational database schema
